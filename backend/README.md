# Wirewave

## Описание проекта
**Wirewave** — это интернет-магазин с возможностью управления товарами, атрибутами и пользователями. Проект построен с использованием **Spring Boot**, **Hibernate** и **PostgreSQL**, поддерживает аутентификацию через **JWT**, а также предоставляет API, документированное с помощью **Swagger**.

## Технологический стек
- **Язык программирования:** Java 8
- **Фреймворк:** Spring Boot 5
- **База данных:** PostgreSQL (используется H2 для тестирования)
- **ORM:** Hibernate
- **Безопасность:** Spring Security, JWT
- **API-документация:** OpenAPI (Swagger)
- **Сборка и управление зависимостями:** Maven
- **Контейнеризация:** Docker
- **Тестирование:** JUnit, Mockito

## Архитектура проекта
Проект построен по принципу многослойной архитектуры:
- **Контроллеры** — обработка HTTP-запросов
- **Сервисы** — бизнес-логика
- **Репозитории** — работа с базой данных
- **Модели данных** — сущности базы данных
- **Конфигурация безопасности** — Spring Security + JWT
- **Логирование** — подробные логи запросов и операций через SLF4J

## Установка и запуск
### 1. Локальный запуск (без Docker)
```sh
mvn clean install
java -jar target/wirewave-0.0.1-SNAPSHOT.jar
```

### 2. Запуск через Docker
```sh
docker-compose up --build
```

### 3. Переменные окружения и конфигурация
Основные настройки находятся в файлах:
- `application.properties`
- `application-test.properties` (для тестов)
- `application-secret.properties` (для конфиденциальных данных)

**Важно:** Перед запуском убедитесь в наличии файла `application-secret.properties` в корневой директории `backend` с корректными настройками базы данных и параметрами JWT.

### Порядок запуска
1. Склонировать репозиторий:
```sh
git clone https://github.com/Brendow271/WireWave.git
```
2. Перейти в директорию backend:
```sh
cd WireWave/backend
```
3. Убедиться в наличии файла `application-secret.properties` с необходимыми настройками.
4. Запустить проект одной из описанных команд (через Maven или Docker).

## API Документация
Проект предоставляет REST API, документированное с использованием **Swagger**.

- Swagger UI: [http://localhost:8080/swagger-ui.html](http://localhost:8080/swagger-ui.html)
- Полный список эндпоинтов доступен в разделе `v3/api-docs`

### Примеры запросов:
#### Получение списка атрибутов
```http
GET /api/attributes
```
#### Создание атрибута (доступно для администраторов)
```http
POST /api/attributes
Content-Type: application/json
{
  "name": "Цвет",
  "type": "string"
}
```
#### Авторизация пользователя
```http
POST /api/auth/login
Content-Type: application/json
{
  "username": "admin",
  "password": "password"
}
```

## Задание
### Функциональные требования
- Каталог товаров с древовидной структурой и возможностью отображения товаров из подкатегорий.
- Аутентификация пользователей с проверкой email/пароля.
- Регистрация новых пользователей с проверкой уникальности email.
- Корзина с возможностью изменения количества товаров и оформлением заказа.

### Реализация бизнес-логики
- Проверка наличия товара перед добавлением в корзину.
- Различные роли пользователей (администратор, обычный пользователь).

### Механизмы фильтрации
- Фильтрация товаров по характеристикам и диапазонам значений.
- Возможность сортировки товаров по цене.

### Безопасность
- Использование JWT токенов для аутентификации.
- Проверка доступа к административным операциям через аннотации Spring Security.

## Тестирование
- Модульное тестирование бизнес-логики.
- Интеграционное тестирование работы контроллеров.
- Проверка безопасности аутентификации.
- Тестирование корректности взаимодействия с базой данных.

### Запуск тестов
Для запуска всех тестов используйте команду:
```sh
mvn test
```
Для запуска конкретного теста:
```sh
mvn -Dtest=ClassName test
```

### Конфигурация тестов
- Используется база данных H2 в памяти (application-test.properties).
- Логирование тестов доступно при включении соответствующих уровней логов.
- Тесты безопасности используют Spring Security Test.

### Основные сценарии тестирования
1. Проверка регистрации, авторизации и выхода пользователей.
2. Проверка добавления товаров в корзину и корректности оформления заказа.
3. Проверка работы фильтров и сортировки каталога.
4. Проверка корректной работы административных функций.
